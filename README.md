# 📚 BookRidge

## 📌 프로젝트 소개

BookRidge는 Java를 약 3주 학습한 이후, 부트캠프 과정에서 처음으로 구현한 도서 관리 시스템입니다.
콘솔 기반 환경에서 도서 관리, 회원 관리, 대출/반납 기능을 구현하며 **객체지향 구조와 데이터 흐름을 이해하는 데 목적**을 두었습니다.

---

## 🗓 개발 기간

* 2025년 (부트캠프 과정 중)
* Java 학습 약 3주 후 진행

---

## 🛠 기술 스택

* **Language**: Java
* **Database**: Oracle DB (JDBC)
* **Build/IDE**: IntelliJ IDEA

---

## 📂 프로젝트 구조

```
BookRidge
 ┣ src
 ┃ ┣ BookRidgeMain      # 메인 실행 및 전체 흐름 제어
 ┃ ┣ BookRidgeDAO       # DB 접근 로직 (JDBC)
 ┃ ┣ BookRidgeDTO       # 데이터 객체 (Book, Member 등)
 ┃ ┗ BookRidgeUtility   # 기능별 유틸리티 (도서 관리 등)
```

---

## ✨ 주요 기능

### 👤 회원 기능

* 회원가입 / 로그인
* 회원 정보 조회

### 📚 도서 기능

* 도서 등록 / 수정 / 삭제
* 도서 검색 (이름, 카테고리)
* 도서 목록 조회 및 페이징 처리

### 🔄 대출 시스템

* 도서 대출 / 반납
* 예약 기능
* 대출 상태 관리

### 🏢 기타 기능

* 희망 도서 신청
* 공지사항 조회
* 스터디룸 기능

---

## 🚀 실행 방법

1. 프로젝트 클론

```
git clone https://github.com/SanghyeokLee-KR/BookRidge.git
```

2. Oracle DB 설정

* DB 연결 정보는 `DBC.java`에서 설정

3. 실행

* `BookRidgeApplication.java` 실행

---

## 📖 배운 점

* Java 기본 문법 및 객체지향 개념 적용
* DTO / DAO 구조를 통한 역할 분리
* JDBC를 이용한 DB 연동 방식 이해
* 콘솔 기반 서비스 흐름 설계 경험

---

## ⚠ 아쉬운 점 및 개선 방향

* DB 연결 정보가 코드에 하드코딩됨 → 설정 분리 필요
* 일부 클래스에 로직이 집중되어 구조 개선 필요
* 제네릭 미사용으로 타입 안정성 부족
* 예외 처리 및 코드 리팩토링 필요
* UI가 콘솔 기반이라 사용자 경험이 제한적
